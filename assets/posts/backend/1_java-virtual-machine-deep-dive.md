## 一、JVM内存结构

Java虚拟机的内存结构可以分为以下几个主要区域：

### 1.1 堆内存（Heap）

堆是Java虚拟机管理的最大一块内存区域，用于存储对象实例和数组。

```java
public class HeapExample {
    public static void main(String[] args) {
        // 对象存储在堆中
        Object obj = new Object();
        // 数组也存储在堆中
        int[] array = new int[100];
    }
}
```

**堆内存特点：**
- 线程共享区域
- 垃圾回收的主要区域
- 通过 `-Xms` 和 `-Xmx` 参数调整大小

### 1.2 方法区（Method Area）

方法区用于存储已被虚拟机加载的类信息、常量、静态变量等。

### 1.3 程序计数器（Program Counter Register）

每个线程都有独立的程序计数器，用于记录当前线程执行的字节码位置。

### 1.4 Java栈（Java Stack）

每个线程对应一个Java栈，用于存储栈帧：

```
┌─────────────────────────────────────────────────────┐
│              Java Stack (线程私有)                   │
├─────────────────────────────────────────────────────┤
│  Stack Frame 1 (main方法)                           │
│  ┌─────────────────────────────────────────────┐    │
│  │  Local Variables (局部变量表)                 │    │
│  │  Operand Stack (操作数栈)                    │    │
│  │  Frame Data (帧数据)                         │    │
│  └─────────────────────────────────────────────┘    │
├─────────────────────────────────────────────────────┤
│  Stack Frame 2 (调用的方法)                         │
│  ┌─────────────────────────────────────────────┐    │
│  │  Local Variables                           │    │
│  │  Operand Stack                             │    │
│  │  Frame Data                                │    │
│  └─────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────┘
```

## 二、垃圾回收机制

### 2.1 引用计数法

**原理：** 每个对象维护一个引用计数器，当引用数为0时回收。

**缺点：** 无法解决循环引用问题。

### 2.2 可达性分析算法

**原理：** 从GC Roots出发，标记所有可达对象，未标记的对象即为垃圾。

**GC Roots包括：**
- 虚拟机栈中引用的对象
- 方法区中静态属性引用的对象
- 方法区中常量引用的对象
- Native方法栈中引用的对象

### 2.3 常见垃圾收集器

| 收集器 | 类型 | 特点 |
|--------|------|------|
| Serial | 串行 | 单线程，适合小型应用 |
| Parallel | 并行 | 多线程，高吞吐量 |
| CMS | 并发 | 低延迟，响应时间优先 |
| G1 | 分区 | 兼顾吞吐量和延迟 |

## 三、性能调优实践

### 3.1 JVM参数配置

```bash
java -Xms2G -Xmx4G -XX:+UseG1GC -XX:MaxGCPauseMillis=200 MyApp
```

**参数说明：**
- `-Xms`：初始堆大小
- `-Xmx`：最大堆大小
- `-XX:+UseG1GC`：使用G1收集器
- `-XX:MaxGCPauseMillis`：最大GC停顿时间

### 3.2 内存泄漏检测

使用工具检测内存泄漏：

```bash
# 使用jmap生成堆转储
jmap -dump:format=b,file=heap.hprof <pid>

# 使用jhat分析堆转储
jhat heap.hprof
```
